2. **Question** : À quoi sert l'attribut 'href' dans une balise 'a' sur un site web ?  
   **Réponse** : Il indique l'adresse ou la page vers laquelle le lien doit emmener l'utilisateur.

3. **Question** : Que fait le premier sélecteur dans l'exemple donné ?  
   **Réponse** : Il cherche les liens qui vont exactement à 'https://google.com' et leur met un contour rouge.

4. **Question** : Comment le deuxième sélecteur trouve-t-il les éléments à cibler ?  
   **Réponse** : Il cherche tous les liens qui contiennent le mot 'google' dans leur adresse et leur met un contour vert.

   **Réponse** : Le premier cible les liens exacts, tandis que le deuxième cible tous les liens incluant 'google'.

6. **Question** : Peut-on utiliser ce type de sélecteur pour mettre en évidence des liens spécifiques sur une page web ?  
   **Réponse** : Oui, on peut utiliser ces sélecteurs pour mettre en évidence des liens selon leur adresse.

7. **Question** : Que signifie l'astérisque (*) dans le sélecteur '[href*="google"]' ?  
   **Réponse** : Il signifie que l'on cherche n'importe quelle partie de l'adresse qui contient 'google'.

8. **Question** : Quel est l'effet d'appliquer 'outline: 1px solid red' à un élément ?  
   **Réponse** : Cela ajoute un contour rouge solide d'un pixel autour de l'élément.

Ces questions et réponses sont formulées pour faciliter la compréhension des sélecteurs CSS par attribut et de leur utilisation pratique dans la personnalisation des styles de page web, tout en restant accessibles à un jeune public.

1. **Question** : Qu'est-ce qu'un "CSS Reset" ?  
   **Réponse** : C'est une manière de rendre les styles de base de tous les navigateurs identiques pour que notre site apparaisse de la même manière partout.

2. **Question** : À quoi sert le fichier index.css dans la création d'un site web ?  
   **Réponse** : Il contient les styles de base et les réinitialisations CSS pour que le site ait un point de départ uniforme.

3. **Question** : Où peut-on trouver le code HTML spécifique pour chaque partie d'un site web ?  
   **Réponse** : Dans les fichiers de composants correspondants à chaque partie.

4. **Question** : Que fait-on dans le fichier app.css ?  
   **Réponse** : On organise la disposition des différentes parties du site, comme où les choses doivent aller sur la page.

5. **Question** : Pourquoi sépare-t-on les styles de mise en page dans un fichier différent comme app.css ?  
   **Réponse** : Pour garder notre code organisé et rendre plus facile la modification de la disposition du site sans changer son apparence générale.

6. **Question** : Qu'est-ce qu'un "composant" dans la création d'un site web ?  
   **Réponse** : C'est une partie du site web, comme un bouton ou un menu, que l'on peut utiliser plusieurs fois.

7. **Question** : Comment peut-on rendre notre site web cohérent sur différents navigateurs ?  
   **Réponse** : En utilisant un CSS Reset dans le fichier index.css pour uniformiser les styles de base.

Ces quiz sont conçus pour aider les enfants à comprendre les bases de la construction d'un site web en termes simples, en se concentrant sur l'organisation et la structure sans entrer dans les détails techniques complexes.
