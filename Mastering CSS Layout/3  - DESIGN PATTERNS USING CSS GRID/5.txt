   **Réponse**: "vh" signifie "hauteur de l'viewport" et est utilisé pour définir la taille d'un bloc par rapport à la hauteur de l'écran.

3. **Question**: Qu'est-ce que `[data-cover]` fait dans le code CSS ?
   **Réponse**: Il applique le modèle de conception "cover" à un bloc pour qu'il couvre entièrement l'espace vertical disponible.

4. **Question**: Pourquoi utiliser `--gutter: initial;` dans notre CSS ?
   **Réponse**: Pour définir un espace par défaut entre les éléments dans le bloc, qui peut être ajusté si nécessaire.

5. **Question**: Comment le code CSS centre-t-il le contenu verticalement ?
   **Réponse**: En utilisant `align-self: center;` sur le contenu qui doit être centré verticalement à l'intérieur du bloc "cover".

6. **Question**: Qu'est-ce que `display: grid;` fait dans le modèle "cover" ?
   **Réponse**: Il organise le contenu du bloc en grille, ce qui aide à positionner et à aligner les éléments plus facilement.

7. **Question**: À quoi sert `min-block-size: 100%;` dans notre code ?
   **Réponse**: Cela assure que le bloc "cover" ait une taille minimale qui couvre 100% de la hauteur de son conteneur parent.

8. **Question**: Pourquoi utiliser `gap: var(--gutter, 0px);` ?

    **Réponse**: En ajoutant l'attribut `[data-cover-centered]` sur l'élément que l'on souhaite centrer.

11. **Question**: Quel est le rôle de `var(--gutter, 0px);` dans la création d'espaces ?
    **Réponse**: Il utilise une variable CSS pour l'espace, qui peut être ajustée facilement, avec une valeur de retour de 0 pixel.

12. **Question**: Pourquoi est-ce important de centrer le contenu dans une conception web ?
    **Réponse**: Pour rendre la page plus agréable visuellement et plus facile à lire pour l'utilisateur.

13. **Question**: Que signifie avoir un bloc qui "couvre 100% de son conteneur parent" ?
    **Réponse**: Cela signifie que le bloc s'étend sur toute la hauteur disponible de l'espace où il se trouve.
